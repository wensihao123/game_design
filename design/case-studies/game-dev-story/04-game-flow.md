## 1. 文档目的

本文件分析《游戏发展国》的完整游戏流程，重点回答：

```text
玩家从开局到后期经历了哪些阶段；
不同阶段的主要目标和压力如何变化；
核心循环在早期、中期和后期有什么不同；
系统如何逐步开放；
平台、员工、资金、内容和公司规模如何共同推动流程；
为什么游戏能够让同一套项目循环在多个阶段保持不同意义；
自研主机为什么是流程终局；
中后期为什么容易从经营转向优化和记录挑战。
```

## 2. 总体流程概览

《游戏发展国》的整体流程可以概括为：

```text
小型公司起步
→ 依靠外包和低成本项目生存
→ 建立第一套稳定开发方法
→ 培养员工并形成成功作品
→ 扩大办公室和团队规模
→ 进入更大平台和更高预算项目
→ 建立品牌、系列和奖项优势
→ 培养高阶职业和顶级人才
→ 开发自有主机
→ 进入记录挑战和全解锁阶段
```

### 2.1 流程不是线性关卡

游戏没有传统章节选择。

它主要通过：

- 时间推进；
- 平台出现；
- 公司条件；
- 员工成长；
- 资金规模；
- 内容解锁；
- 办公室扩张；
- 高阶目标；

自然形成阶段变化。

### 2.2 阶段变化的本质

不同阶段仍然使用同一个核心循环：

```text
立项
→ 开发
→ 评价
→ 销售
→ 再投资
```

但每个阶段改变：

- 可承担的风险；
- 可使用的员工；
- 可选择的平台；
- 可使用的内容；
- 失败的后果；
- 成功的上限；
- 长期目标。

## 3. 游戏流程的四种驱动力

### 3.1 经济驱动

```text
现金不足
→ 需要项目
→ 项目成功
→ 获得更多现金
→ 解锁更大投入
```

### 3.2 人才驱动

```text
员工成长
→ 项目能力提升
→ 更高质量作品
→ 更高收益
→ 进一步培养员工
```

### 3.3 内容驱动

```text
职业与训练
→ 解锁新类型和题材
→ 增加项目组合
→ 形成新探索目标
```

### 3.4 行业驱动

```text
新平台出现
→ 市场迁移
→ 旧策略失效
→ 玩家重新投资
```

这四种驱动力共同推动流程，而不是只有单一经验条。

---

## 4. 阶段划分

本分析将《游戏发展国》的整体流程划分为六个阶段：

1. 创业生存；
2. 建立方法；
3. 稳定增长；
4. 规模扩张；
5. 行业领先；
6. 自研平台与终局。

这些阶段不是游戏内明确命名的章节，而是根据系统压力和玩家目标变化进行的设计划分。

---

# 5. 阶段一：创业生存

## 5.1 初始状态

玩家通常面对：

- 员工人数少；
- 员工能力低；
- 可选内容有限；
- 现金储备小；
- 平台选择少；
- 固定工资相对明显；
- 对系统规律了解不足。

### 5.2 核心目标

```text
让公司持续运转。
```

### 5.3 主要行为

- 招募基础员工；
- 承接外包；
- 开发低成本作品；
- 控制工资；
- 避免过早进入高成本平台；
- 学习项目流程；
- 观察评论和销量。

### 5.4 主要压力

#### 现金流压力

任何项目失败都可能推迟：

- 招聘；
- 平台授权；
- 训练；
- 扩张。

#### 信息压力

玩家不知道：

- 哪些组合好；
- 哪些员工值得培养；
- 如何分配研究点；
- 什么平台更值得进入。

#### 人员压力

团队能力不足，项目质量波动较大。

### 5.5 外包的流程作用

外包在此阶段非常重要。

```text
现金不足
→ 接外包
→ 获得稳定收入
→ 支付工资
→ 再尝试自研项目
```

它相当于：

```text
开局安全网。
```

### 5.6 自研项目的意义

第一批自研游戏承担：

- 教学；
- 风险体验；
- 组合探索；
- 评论揭晓；
- 销量体验；
- 首次品牌积累。

### 5.7 阶段情绪

- 谨慎；
- 好奇；
- 紧张；
- 小成功带来的明显满足。

### 5.8 阶段完成信号

玩家进入下一阶段，通常意味着：

- 公司不会因一次普通失败立刻陷入危机；
- 已经完成数个自研项目；
- 有基本现金储备；
- 开始掌握部分组合；
- 员工开始形成成长方向。

---

# 6. 阶段二：建立方法

## 6.1 阶段状态

玩家不再完全陌生，但仍没有强大公司能力。

### 6.2 核心目标

```text
找到一套可以稳定运转的开发方法。
```

### 6.3 主要行为

- 重复部分可靠组合；
- 尝试新类型和题材；
- 培养核心员工；
- 学习职业与训练；
- 规划平台授权；
- 开始比较内部制作与外包；
- 尝试宣传；
- 积累粉丝。

### 6.4 主要决策

#### 专精还是探索

玩家开始判断：

```text
继续做熟悉组合
```

还是：

```text
尝试新组合和新内容。
```

#### 培养还是招聘

- 培养当前员工更便宜但需要时间；
- 招聘新员工更快但增加工资。

#### 现金储备还是投资

- 培训；
- 转职；
- 新平台；
- 宣传；
- 更高成本项目。

### 6.5 系统开放感

此阶段通常会让玩家逐步认识：

- 员工职业；
- 类型和题材等级；
- 组合相性；
- 平台用户量；
- 宣传作用；
- 粉丝作用；
- 公司扩张条件。

### 6.6 反馈变化

玩家开始从：

```text
看单次项目是否成功
```

转向：

```text
看自己的方法是否可重复。
```

### 6.7 阶段情绪

- 学习；
- 试验；
- 初步掌控；
- 对成长路径产生规划。

### 6.8 阶段完成信号

- 有一组稳定核心员工；
- 可以连续完成自研项目；
- 已形成部分高评价组合；
- 能够承担中等项目成本；
- 开始考虑扩大团队或进入更大平台。

---

# 7. 阶段三：稳定增长

## 7.1 阶段状态

公司已经从“活下来”转向“持续变强”。

### 7.2 核心目标

```text
把偶然成功变成稳定成功。
```

### 7.3 主要行为

- 集中培养核心员工；
- 规划多职业成长；
- 提高类型和题材等级；
- 使用更高成本平台；
- 增加宣传；
- 建立系列作品；
- 追求更高评论；
- 积累粉丝和现金。

### 7.4 主要压力变化

#### 现金压力下降

现金仍重要，但不再是唯一问题。

#### 人才压力上升

玩家开始意识到：

- 团队能力决定作品上限；
- 员工职业路径需要长期规划；
- 高薪员工会增加固定成本。

#### 内容压力上升

玩家希望解锁：

- 新类型；
- 新题材；
- 新组合；
- 高阶职业。

### 7.5 项目循环变化

早期项目是：

```text
能否赚钱。
```

中期项目变成：

```text
能否达到更高评价并形成长期品牌。
```

### 7.6 粉丝开始发挥作用

成功作品产生的粉丝提高后续项目市场基础。

这使增长进入复利阶段。

### 7.7 平台策略变化

玩家不再只选最便宜的平台，而会考虑：

- 用户量；
- 生命周期；
- 成本；
- 是否值得长期开发；
- 是否为未来项目铺路。

### 7.8 阶段情绪

- 扩张；
- 成功复利；
- 团队成形；
- 对项目结果更有信心。

### 7.9 阶段完成信号

- 高评价不再完全依赖运气；
- 公司有较强现金储备；
- 核心员工明显成长；
- 粉丝开始形成持续优势；
- 办公室扩张或团队扩张成为现实目标。

---

# 8. 阶段四：规模扩张

## 8.1 阶段状态

公司开始拥有更多员工、更高工资和更大项目能力。

### 8.2 核心目标

```text
把个人能力转化为组织能力。
```

### 8.3 主要行为

- 扩大办公室；
- 增加员工数量；
- 调整团队结构；
- 招聘高潜或高阶员工；
- 培养多个职业；
- 承担高成本项目；
- 进入主流平台；
- 提高宣传规模；
- 冲击高销量。

### 8.4 扩张的双重作用

办公室扩张带来：

- 更多工位；
- 更高总产出；
- 更大团队；
- 更强视觉成长。

同时也带来：

- 更高工资；
- 更高固定成本；
- 更复杂培养需求；
- 更大现金流要求。

### 8.5 团队结构变化

早期关注：

```text
有没有人。
```

此阶段关注：

```text
谁应该留下，谁应该培养，谁应该被替换。
```

### 8.6 员工成为组织资产

玩家开始形成：

- 核心员工；
- 专项员工；
- 培养中的员工；
- 高薪顶级员工；
- 可能被替换的员工。

### 8.7 项目策略变化

项目开始承担更多目标：

- 赚钱；
- 提高粉丝；
- 冲击奖项；
- 培养员工；
- 提升类型等级；
- 为系列铺路；
- 为平台迁移准备。

### 8.8 失败后果变化

单次普通项目失败通常不会让公司破产。

失败主要造成：

- 增长减速；
- 错过扩张机会；
- 延迟平台投资；
- 浪费高额成本。

### 8.9 阶段情绪

- 规模感；
- 管理感；
- 组织优化；
- 成功稳定化。

### 8.10 阶段完成信号

- 公司拥有较完整团队；
- 现金流能承担高额固定成本；
- 进入高用户平台；
- 能稳定冲击高评分和高销量；
- 开始追求行业奖项和顶级人才。

---

# 9. 阶段五：行业领先

## 9.1 阶段状态

公司已经具备：

- 高能力员工；
- 高粉丝；
- 高现金储备；
- 多内容解锁；
- 多平台经验；
- 稳定成功方法。

### 9.2 核心目标

```text
从成功公司变成行业标杆。
```

### 9.3 主要行为

- 追求高评论记录；
- 追求高销量记录；
- 获得奖项；
- 建立成功系列；
- 招募或培养顶级员工；
- 完成高阶职业路线；
- 准备自研主机；
- 完成剩余解锁。

### 9.4 压力从生存转向优化

早期压力来自：

- 钱不够；
- 员工太弱；
- 平台太贵。

后期压力来自：

- 能否更高；
- 能否更快；
- 能否全解锁；
- 能否打破记录；
- 能否完成终局条件。

### 9.5 奖项的流程作用

奖项提供：

- 高阶目标；
- 行业认可；
- 项目成功以外的评价；
- 年度节奏；
- 记录。

### 9.6 系列的流程作用

系列允许过去成功继续影响新项目。

### 9.7 平台判断趋于成熟

玩家已经理解平台生命周期。

此时平台选择更多是：

- 效率优化；
- 市场最大化；
- 终局准备。

### 9.8 阶段情绪

- 权力感；
- 掌控感；
- 追求极限；
- 收集和完成。

### 9.9 阶段完成信号

- 公司具备自研主机所需条件；
- 多数基础系统已被掌握；
- 资源压力明显下降；
- 玩家目标转向终局身份或纪录。

---

# 10. 阶段六：自研平台与终局

## 10.1 阶段状态

玩家已经不满足于依附外部平台。

### 10.2 核心目标

```text
开发自己的主机，
从内容开发商升级为平台拥有者。
```

### 10.3 前置积累

通常需要：

- 大量资金；
- 高阶员工；
- 特定职业能力；
- 公司规模；
- 长期发展。

### 10.4 自研主机的流程作用

自研主机是多个系统的汇合点：

```text
资金
+
员工
+
职业
+
公司规模
+
行业经验
```

### 10.5 身份跃迁

此前玩家：

```text
选择市场。
```

之后玩家：

```text
创造市场。
```

### 10.6 终局后的主要目标

- 为自家平台开发作品；
- 提高主机表现；
- 追求极限销量；
- 追求最高评价；
- 完成奖项；
- 完成员工培养；
- 完成全解锁；
- 打破历史记录。

### 10.7 终局问题

自研主机之后，游戏缺少同等强度的新系统层级。

因此流程从：

```text
结构成长
```

逐渐转向：

```text
数值优化和自我挑战。
```

## 12. 系统开放节奏

### 12.1 开局开放

玩家首先接触：

- 现金；
- 员工；
- 外包；
- 自研项目；
- 类型；
- 题材；
- 基础平台；
- 评论；
- 销量。

### 12.2 中期开放

逐步强化：

- 训练；
- 职业；
- 转职；
- 新平台；
- 宣传；
- 粉丝；
- 办公室扩张；
- 更多组合。

### 12.3 后期开放

- 高阶职业；
- 顶级员工；
- 奖项目标；
- 续作；
- 自研主机；
- 极限记录。

### 12.4 开放原则

```text
先让玩家理解项目循环，
再增加员工和内容复杂度，
最后开放组织和平台级目标。
```

## 13. 引导结构

### 13.1 隐式教学

游戏主要通过：

- 少量初始选项；
- 低成本项目；
- 外包；
- 评论；
- 销量；
- 逐步解锁；

让玩家理解系统。

### 13.2 结果教学

玩家通过结果学习：

```text
什么组合更好；
员工能力是否足够；
平台是否值得；
宣传是否有效。
```

### 13.3 失败教学

失败不会立即终止，因此可成为低风险学习。

### 13.4 节奏教学

系统不是一次性全部开放，而是随公司成长逐渐增加。

## 14. 目标结构

### 14.1 短期目标

- 完成当前阶段；
- 修完 Bug；
- 看评论；
- 达到销量里程碑；
- 支付工资。

### 14.2 中期目标

- 培养员工；
- 买新平台；
- 解锁内容；
- 扩大办公室；
- 建立系列；
- 获得奖项。

### 14.3 长期目标

- 百万销量；
- 顶级团队；
- 高阶职业；
- 自研主机；
- 全解锁；
- 纪录挑战。

### 17.4 风险衰减

## 23. 流程中的软门槛

### 23.1 现金门槛

- 高宣传；
- 招高薪员工；

### 23.2 能力门槛

### 23.3 知识门槛

### 23.4 职业门槛

### 23.5 时间门槛

平台和行业机会随年份出现。

### 23.6 门槛特点

- 办公室容量；

## 33. 中后期流程衰减

### 33.1 资源门槛失效

### 33.3 知识门槛失效

最佳组合和解锁路径已知。

### 33.4 平台门槛简化

玩家已熟悉平台生命周期。

### 33.5 目标门槛减少

自研主机后缺少更高身份层。

### 33.6 流程结果

## 34. 终局结构

### 34.1 硬终局

游戏通常没有严格强制结束。

### 34.2 软终局

由以下目标构成：

- 自研主机；
- 顶级评价；
- 顶级销量；
- 奖项；
- 全解锁；
- 高阶员工；
- 玩家自定纪录。

### 34.3 软终局优点

- 玩家可继续经营；
- 不强制中断；
- 支持长期优化。

### 34.4 软终局缺点

- 完成感依赖玩家自定目标；
- 新结构变化较少；
- 后期重复明显。

## 37. 游戏流程的主要取舍

### 37.1 阶段依赖数值门槛

代价：

### 37.3 强复利

加速成长，但削弱后期经营压力。

### 37.4 软终局

自由度高，但结局仪式较弱。

## 42. 游戏流程结论

### 42.1 结论一：流程由压力变化定义

《游戏发展国》的阶段不是通过章节切换，而是通过：

- 现金；
- 员工；
- 平台；
- 公司规模；
- 长期目标；

的主要压力变化自然形成。

### 42.2 结论二：同一核心循环被多次重新解释

早期项目用于生存。

中期项目用于成长。

后期项目用于记录和行业地位。

### 42.3 结论三：外包保证流程不会轻易中断

它为现金危机和项目失败提供恢复路径。

### 42.4 结论四：办公室和主机是关键身份转折

它们把抽象数值成长转化为可见公司变化。

### 42.5 结论五：平台时代推动外部节奏

即使公司方法成熟，市场仍在变化。

### 42.6 结论六：中后期衰减来自压力消失

当现金、员工和知识都不再构成瓶颈，流程从经营转向重复优化。

### 42.7 结论七：终局需要新的决策层

自研主机完成后，如果没有更高层组织和市场系统，长期动力会依赖记录与全解锁。

```text
员工能力增长

内容逐步解锁
+
平台时代变化
+
身份层级提升
```

最值得借鉴的地方在于：

```text
游戏没有频繁更换核心玩法，
而是不断改变同一项目循环在当前阶段中的意义。
```

这使一套相对简单的循环能够支撑从创业到终局的完整成长旅程。
