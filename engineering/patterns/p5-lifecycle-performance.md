# Patterns P5:生命周期与性能(Lifecycle & Performance)

> **索引归属**:`README.md` §4 中的 P5 册,全专题末册。双面模板见 README。
>
> **本册地位**:与前四册有一个根本差异——**前四册的模式改善结构(收益是可维护性),本册的模式增加结构复杂度以换取运行时性能(收益必须可测量)**。因此本册所有卡片共享一道前置门(§0),反面主角"过早优化"压轴出场并接受一次公正的重审(§6)。

---

## 0. 前置门:证据先行

本册每张卡片的适用信号都隐含同一条第 0 信号:**profiler 证据在手**。无测量的性能模式引入,一律按过早优化处理。固定三问,嵌入每次优化动作:

1. **测了吗?**——Godot profiler / Performance 监视器 / 自建计数器(tick 耗时、分配次数),拿到基线数字
2. **是热点吗?**——该处耗时占比是否值得动(不在前几名的"优化"是自我安慰)
3. **改后复测了吗?**——数字对比进优化台账(改动、前后数字、日期);没有第 3 步的优化等于没做

本册卡片的翻面因此形态特殊:腐化信号大多不是"模式变形",而是**"模式被无证据引入"或"模式自身引入了新 bug 类"**——性能模式的第一风险从来不是不够快,是脏状态、陈旧数据、时序错乱这些正确性损伤。

## 卡片 1:对象池

- **粒度层 / 吸收状态**:系统 / 🟡
- **解决的问题**:高频生灭对象(子弹、粒子、飘字、音效实例)的实例化/释放开销与由此产生的帧尖峰
- **Godot 形态**:预分配 + 停用复用。节点对象的"停用"要做全:移出树或隐藏 + `set_process(false)` + 碰撞关闭——半停用的池成员还在消耗每帧成本(P4 卡片 2 腐化③的池版)
- **适用信号**:① profiler 显示 instantiate/free 进入热点;② 帧尖峰与生成潮汐(开枪、爆炸)在时间轴上重合

──────── 翻面 ────────

- **入口诱惑**:"子弹当然要池化"——货物崇拜的教科书案例:结论被记住了,前提(每秒数百级的生灭)被忘了。每秒三个的东西入池,收益为零,下述风险照单全收
- **腐化信号**:① **脏对象回收**——本卡片的第一风险:复用的子弹带着上一世的速度、tween、信号连接、计时器出场。防线是 **reset 契约**:池化类型必须实现 `reset()`,池在出借时强制调用;reset 遗漏字段的 bug 偶发且与池实现纠缠,极难归因;② 池无上限——泄漏换了个马甲(对象不再释放,只是永远"借出"未还);池必须有容量上限 + 借出计数面板;③ 池化低频对象(诱惑的后果正式立案)
- **不可逆点**:无 reset 契约的池上线后,每个偶发脏状态 bug 都要在"业务逻辑错"与"池没洗干净"之间二选一,排查成本随池化类型数线性增长
- **解药**:reset 契约 + 借还统计为池的标配件(P1 §6b"可观测性是标配"同款裁定);低频对象退回直接 instantiate
- **关联**:P2 卡片 5 翻面②(运行时 duplicate 当原型 = 无池之名的脏对象复用,同病)

## 卡片 2:享元

- **粒度层 / 吸收状态**:对象 / 🟢(Resource 引用共享即引擎原生享元;渲染侧另有 MultiMesh——万株草共享一份网格,实例仅存变换)
- **解决的问题**:海量实例共享不变数据,内存与加载从 O(实例) 降到 O(种类)
- **Godot 形态与实战**:已在两处完整建成——`[FSM:C篇 §4.1]` 三层切分(图/def/ctx)、`[P2卡片3]` 类型对象。本卡片主要是把它们钉进性能视角:**享元是本知识库唯一"默认开启"的性能模式**——因为它同时是正确的结构(def/实例分离),性能收益是白捡的
- **适用信号**:实例 ≫ 种类,且种类数据不变——即 def 体系的成立前提,无需额外证据(前置门唯一豁免,理由如上)

──────── 翻面 ────────

- **入口诱惑**:无(结构自然导向它)——危险全在使用细节
- **腐化信号**:① 共享数据被运行时改写——**本知识库最高频引用的陷阱第三次立案**(`[FSM:C篇 §4.4]` Resource 默认共享、`[P2卡片5]` duplicate 浅复制语义),值得升格为全库级红线:**任何 Resource 字段赋值语句,先问这份 Resource 有几个持有者**;② 该独立变化的数据被享元化——两个建筑该各自升级,却共享同一份可变配置,一改全改
- **不可逆点**:共享写污染是运行时偶发损坏,存档后固化——损坏会被存进玩家档案
- **解药**:def 不可变纪律 + 类型级分工(可变状态一律 RefCounted)`[边界篇 §4.2]`;需要"基于模板的独立副本"时显式 duplicate 并注明深浅
- **关联**:全库引用网的枢纽卡片

## 卡片 3:脏标记

- **粒度层 / 吸收状态**:系统 / 🟡
- **解决的问题**:昂贵的**派生值**(城市总产出 = Σ全部建筑、寻路网格、UI 布局)被高频读取、低频变更——缓存结果,变更时置脏,读取时按需重算
- **Godot 形态**:`_cached + _dirty` 字段对,或聚合系统级的失效集合。管理游戏的聚合统计(总人口、总收支、幸福度)是本卡片的主场
- **适用信号**:① profiler 显示同一计算被重复执行且输入未变;② 读写比悬殊(每帧读、每 tick 才可能变)

──────── 翻面 ────────

- **入口诱惑**:"加个 cache 呗"——缓存是最容易加、最难保正确的东西;每个 cache 都是一份需要手动同步的冗余真相
- **腐化信号**:① **失效遗漏**——某条写路径忘了置脏,产出陈旧数据 bug:值是错的但看起来合理,测试难捕获,玩家先于你发现。防线是结构性的:**被缓存数据的写路径必须收拢到唯一 setter**(置脏藏在 setter 里,新写路径自动继承)——`[边界篇 §2]` 接口纪律在此兑现真金白银的回报;② 脏标记级联网(A 脏导致 B 脏导致 C 脏)无声明地生长——失效传播图必须显式(注册依赖关系,置脏自动传播),口头约定的级联必漏;③ 为廉价计算加缓存(前置门失守)
- **不可逆点**:多份缓存 + 分散写路径 + 隐式级联三者齐备后,任何数据不一致都无法定位起点
- **解药**:写路径唯一化;传播显式注册;**拿不准就粗粒度**——整体重算一次的成本,通常远低于细粒度失效遗漏一次的排查成本
- **关联**:`[边界篇]` 接口边界(本卡片是其性能红利的直接案例)

## 卡片 4:空间分区

- **粒度层 / 吸收状态**:系统 / 🟢半(物理引擎自带 broad-phase——Area2D/ShapeCast 的邻域查询已是分区加速;**自建**仅当查询不走物理)
- **解决的问题**:"谁在附近"从遍历全体 O(N²) 降为查桶 O(1)~O(logN)
- **Godot 形态**:三档——表现/物理层用引擎自带(Area2D 感知范围);**规则层网格游戏几乎免费**:格子地图本身就是分区,"相邻建筑加成"查询就是查字典 `{cell: entity_id}`,不需要任何额外结构;自由坐标 + 海量动态实体才需要自建(均匀网格优先,四叉树是最后手段)
- **适用信号**:① profiler 显示邻域查询热点且呈 O(N²) 增长;② 规则层需要空间查询而不该碰物理(见翻面①)

──────── 翻面 ────────

- **入口诱惑**:两个方向——"Area2D 现成的,规则探测也用它";"四叉树听起来专业,先建一棵"
- **腐化信号**:① **规则查询走物理引擎**——规则依赖物理帧时序与物理伺服(headless 要拉起物理服务器、查询结果晚一物理帧),P4 三层时间的跨层违规:规则层查询必须用规则层数据结构;② 为基本静止或数量 <100 的对象维护四叉树——结构维护成本(每次移动更新树)超过被优化的查询本身;③ 分区粒度与查询半径失配(格子远大于查询半径 → 每查邻近 9 桶全是无关对象)
- **不可逆点**:规则与物理伺服的耦合蔓延后,headless 测试与确定性回放同时报废——修复要重写全部空间查询
- **解药**:规则层查 grid dict;动态海量才升树;粒度对齐典型查询半径
- **关联**:P4 §0 三层时间(本卡片翻面①是其空间版)

## 卡片 5:双缓冲

- **粒度层 / 吸收状态**:系统 / 🟡
- **解决的问题**:一批代码要**读**的数据正被同一轮次的代码**写**——写下一版、读上一版、轮次边界交换,读写彻底分离
- **Godot 形态与实战**:`[FSM:C篇 §4.3b]` 快照读是它的近亲(快照 = 按需单向缓冲);感知黑板的"事实区"本质是感知系统与 B 机之间的缓冲 `[FSM:B篇]`。完整双缓冲的主场是**元胞自动机式模拟**:火灾蔓延、污染扩散、人口流动——全体格子必须基于同一份旧世界演化,否则先算的格子影响后算的(结果依赖遍历顺序,`[FSM:C篇 §4.3a]` 的病灶)
- **适用信号**:① 全体元素需基于同一旧状态同时演化(顺序无关性是正确性需求,不只是性能);② 读方与写方节拍不同(渲染读、tick 写)

──────── 翻面 ────────

- **入口诱惑**:"稳妥起见都拷一份"——快照习惯泛化成无预算的全量拷贝
- **腐化信号**:① 交换时机无主——谁负责 swap、在哪个边界 swap 说不清;swap 必须钉在 tick/阶段边界上(`[FSM:DEF篇 §5]` 阶段序列是它唯一的家);② **半缓冲**:同一数据集一半走缓冲一半直写——读方看到新旧混合的世界,比不缓冲更糟(不缓冲至少错得一致);③ 每 tick 全量深拷贝大结构且无测量(前置门)
- **不可逆点**:半缓冲的混合读扩散后,"这个值是哪个版本的"成为每处读取的悬案
- **解药**:缓冲以**数据集**为单位全有或全无;swap 归阶段边界;大结构用交换引用(两份常驻,交换指针)而非逐次拷贝
- **关联**:`[FSM:C篇 §4.3]`(本卡片是两阶段结算的通用化)

## 6. 反面专题:过早优化的公正重审

**原判**:"过早优化是万恶之源"——Knuth 原文的语境常被裁掉:他说的是**约 97% 的小效率不值得追逐,但关键的 3% 必须抓住**。流行版把它读成"不要优化",错得和反面一样远。

**游戏开发需要的修正**:把性能决策分成两类,待遇截然相反——

| | 容量规划(架构级) | 微优化(实现级) |
|---|---|---|
| 例 | tick 制还是每帧结算、实体规模上限、def/实例分离、集中还是分散更新 | 池化某类对象、给某聚合加脏标记、某查询建分区 |
| 何时决策 | **提前**——立项/架构期 | **滞后**——profiler 出示证据后 |
| 决策成本 | 低(画架构时顺手) | 各卡片的复杂度税 |
| 更改成本 | 极高(P4 卡片 1/4 不可逆点:数值在错误基底上调平后) | 低(局部改动) |
| 判据 | 目标规模的量级估算(千级建筑?百级 AI?) | §0 三问 |

**"过早优化是万恶之源"只适用于右列**。左列恰恰相反——容量规划的"过晚"才是万恶之源:tick 制没提前定,后期从每帧结算迁移等于重做;实体规模没估量级,def/实例分离就不会做,享元红利错过。本知识库前面全部专题的架构建议,本质上都是左列的容量规划——它们看起来"还没测就定了",但那正是它们该被决定的时机。

**流程纪律**(右列专用):测 → 定位 → 改 → 复测 → 记台账。台账(改动/前后数字/日期)防两件事:无效优化的自我安慰,和三个月后"这段怪代码为什么存在"的考古。**没有台账记录的性能复杂度,重构时一律视为可删。**

## 7. 裁决速查表

| 症状(profiler 语言) | 卡片 | 前提 |
|---|---|---|
| instantiate/free 热点、生成潮汐尖峰 | 对象池 | reset 契约同步交付 |

## 8. AI 代理协作约定(供 CLAUDE.md 摘录)

1. 引入本册任何模式前必须附 profiler 证据与基线数字;无证据的性能改动不予通过(享元/def 分离除外,它是结构默认)。
2. 对象池交付必须含 reset 契约与借还统计;脏标记交付必须先完成写路径唯一化;双缓冲的 swap 必须落在声明的阶段边界。
3. 规则层空间查询禁止经物理引擎;网格游戏优先使用地图本身作为分区。
4. 每次性能优化提交附台账条目(改动/前后数字);发现无台账的性能复杂度,标注为可删候选。
5. 架构期的容量规划问题(tick 制、规模上限、更新集中度)不受"过早优化"约束,按 §6 左列提前决策并记录量级假设。

## 全专题收束(Patterns P1–P5)

五册完成。三条贯穿线,作为整个 Patterns 专题的浓缩:

1. **每个问题域都是一条光谱,裁决都是找位置**——通信的隐式↔显式(P1/P3)、变体的代码↔数据(P2)、时间的三层节拍(P4)、优化的提前↔滞后(P5)。模式选型没有"最好",只有"这个需求在光谱上的坐标"
2. **可观测性是模式的标配件,不是可选项**——事件日志(P1)、借还统计(P5)、优化台账(P5)、组合根即依赖图(P3):每个引入间接性的模式,都必须同时交付看穿这层间接性的工具
3. **最贵的错误都是"正确模式 + 错误层"**——Area2D 做规则查询、autoload 装规则状态、await 编排规则流程、`_process` 跑规则结算:模式本身无罪,跨层即腐化。判层三问始终是:进不进存档?headless 下存不存在?谁的需求会改它?

至此,知识库四专题(SOLID 原则 → 模块边界 → FSM 纵深 → Patterns 工具箱)结构闭合。各文档"AI 代理协作约定"节的汇编、以及两个挂载点(`fsm-f-advanced-interrupts.md`、`patterns-ecs-vs-nodes.md`)留待需要时进行。
