# Patterns P2:数据与行为组织(Data & Behavior)

> **索引归属**:`README.md` §4 中的 P2 册。双面模板、判定三问见 README,不重复。
>
> **本册地位**:问题域的唯一问题——**"变化放在哪里?"** 游戏内容的本质是同一套机制的大量变体(一百种建筑、五十种敌人、三百个道具),代码的形状必须追随变化的形状。本册五张卡片是"用什么承载变体"从代码到数据的完整光谱;反面主角 God Object 与继承深渊,则是变化被塞错地方后的两种坍缩形态。

---

## 0. 问题域光谱:变体的承载物

```
代码 ◄──────────────────────────────────────────────► 数据
继承(变体=子类) → 策略(变体=可换对象) → 组件(变体=部件集合)
                → 类型对象(变体=描述对象) → 数据驱动(变体=纯数据+解释器)
```

越靠左:表达力越强(变体可以是任意代码),生产成本越高(每个变体都是编程动作)。越靠右:生产成本越低(填表即得变体),表达力越受限(只能组合解释器认识的东西)。

裁决的钥匙是看清**差异的形状**:变体之间差的到底是常数、算法、能力集合,还是整段内容?——§7 速查表按此裁决。注意光谱最左端的"继承"没有卡片:`[SOLID篇]` 已裁定其为一层上限的受限工具,本册视其为组件卡片翻面里的病灶而非选项。

## 卡片 1:策略

- **粒度层 / 吸收状态**:对象 / 🟡
- **解决的问题**:同一职责存在多种**算法整体不同**的实现,需要可替换、可独立测试
- **Godot 形态**:RefCounted 策略类,统一接口;或 `[SOLID篇 OCP]` 的 Resource 挂 GDScript 字段(策略以数据形式被引用——本卡片与卡片 4 的接合点)
- **适用信号**:① 变体间的差异无法用参数表达(寻路用 A* 还是流场,不是调个系数的事);② P1 卡片 5 的升格线:同一对象的 Callable 注入点 ≥3 且语义成组

──────── 翻面 ────────

- **入口诱惑**:"开个策略类,以后好扩展"——为参数级差异开类,仪式先于需要
- **腐化信号**:① 策略类批量出现但两两之间只差常数(`FastZombieStrategy` 和 `SlowZombieStrategy` 只差移速)——变体的真实形状是数据,退回类型对象(卡片 3);② 策略反向持有宿主并操控宿主内部——策略应是被调用的纯计算,拿输入给输出;③ 策略里出现 `if host is XXX` 的类型嗅探——接口没抽干净
- **不可逆点**:策略与宿主互相操控后,两者已是隐形单体,拆分等于重写
- **解药**:常数差异 → 卡片 3;宿主操控 → 收窄为纯函数式接口(ctx 进,结果出)
- **关联**:P1 卡片 5(单点简配版);`[FSM:B篇 §5.4]` 效用 AI 的评分函数本质是策略的函数集合形态

## 卡片 2:组件组合

- **粒度层 / 吸收状态**:对象~架构 / 🟢半(场景树即组合的原生土壤:子节点就是组件槽;但**逻辑层组件**仍需手动实践)
- **解决的问题**:实体能力的自由拼装。继承在"能力交叉"面前必然爆炸(会飞的敌人、会飞的宝箱、会喷火的两者)——组合把 M×N 的类爆炸变为 M+N 个部件
- **Godot 形态**:双轨——表现/交互能力用**子节点组件**(Hurtbox、Interactable 挂上即得);规则能力用**纯对象组件**(实体 ctx 持有部件对象列表)。`[SOLID篇 LSP]` 的"组合优先"在此获得实施细则
- **适用信号**:① LSP 信号:子类 override 改变语义;② 能力集合开始**跨类型交叉**(两个不相干的实体类需要同一能力——继承树上没有共同祖先可挂)

──────── 翻面 ────────

- **入口诱惑**:"万物组件化"教条——单一实现、永不复用的功能也被拆成组件,拼装灵活性为零、间接成本全额支付
- **腐化信号**:① 组件之间横向引用(组件 get_node 兄弟组件、互相调用)——组件通信必须经宿主中介(P1 卡片 4 的实体版),横向网一旦织成,组件不再可独立增删;② 粒度过细(HealthComponent 里只有一个 int 和 getter)——按**变化的边界**切组件,不按名词切;③ 以性能之名开始重造 ECS(手写 archetype、组件数组化)——那是另一个架构范式的领域,与节点树的关系微妙,见挂载点 `patterns-ecs-vs-nodes.md`(未建),不在本卡片射程内
- **不可逆点**:横向引用网成形——组件失去可插拔性后,"组合"只剩形式,实为切碎了的 God Object
- **解药**:横向通信收归宿主转发/仲裁;过细组件按变更同频 `[边界篇 §1]` 合并
- **关联**:`[SOLID篇]` 继承一层上限(本卡片是其正面出路);P1 卡片 4

## 卡片 3:类型对象

- **粒度层 / 吸收状态**:对象 / 🟢(**Resource 自定义类就是类型对象的引擎形态**——`class_name BuildingDef extends Resource` 即教科书实现,多数 Godot 开发者天天在用而不自知)
- **解决的问题**:"种类"之间的差异是**一组数值/引用**而非行为——让种类本身成为对象(def),实例持有对种类的引用,新种类 = 新 def 实例,零新类
- **Godot 形态**:def Resource + 实例持 def 引用。`[FSM:C篇 §4.1]` 的三层切分(图/def/ctx)中间层即本卡片
- **适用信号**:① 种类数远大于行为数(一百种建筑,三种行为逻辑);② 新种类的增加是内容动作而非编程动作(策划要自己加怪)

──────── 翻面 ────────

- **入口诱惑**:(本卡片罕见地几乎无诱惑陷阱——它是被低估而非被滥用的模式;危险都在使用细节里)
- **腐化信号**:① def 里开始塞行为开关,消费代码长出 `if def.is_special_a ... elif def.is_special_b` 的分支梯——行为差异冒头了,该给 def 配策略字段(卡片 1 接合点),而不是用 bool 开关模拟多态;② 实例把 def 数据**拷贝**进自己(`[FSM:C篇 §4.1]` 的原罪重现:改 def 不影响存量实例);③ def 之间形成继承链(def 继承 def 继承 def)——数据继承深渊,用**组合引用**(def 持有共享的部件 def)替代
- **不可逆点**:bool 开关梯蔓延进多个消费系统后,每加一种行为要改 N 处分支——OCP 的反面完全体
- **解药**:开关 → 策略字段或封闭 tag `[SOLID篇]`;拷贝 → 引用;数据继承 → 数据组合
- **关联**:卡片 4 的前哨(类型对象 + 封闭分类法 + 解释器 = 数据驱动);`[FSM:G篇 §2.1]` 任务 def 即大规模实战

## 卡片 4:数据驱动 / 表驱动

- **粒度层 / 吸收状态**:架构 / 🟡
- **解决的问题**:变体的生产完全移交内容侧——代码收敛为解释器,变体收敛为数据,新内容零代码
- **Godot 形态**:Resource/JSON + 封闭分类法 + 通用解释器。三个已建成的完整实战:`[SOLID篇]` 效果 tag 体系、`[FSM:G篇]` 条件/效果解释器、`[FSM:A篇 §4.4]` 帧数据与取消窗口
- **适用信号**:① 变体由内容侧批量生产且高频变更;② 变体的差异可被**封闭分类法的组合**表达(能枚举完解释器需要认识的原子)

──────── 翻面 ────────

- **入口诱惑**:"做成可配置的,以后不用改代码"——**软编码(soft-coding)**反模式的开场白:为一次性逻辑、为程序员自己才会改的东西修建配置系统
- **腐化信号**:① 配置里长出编程语言——条件树嵌套十层、配置引用配置、出现变量与循环的模拟物;配置图灵完备之日,就是你拥有一门无调试器、无类型检查、无 IDE 的自制语言之时。正解是承认"这段就是代码",走脚本逃生舱 `[FSM:G篇 §5.1]` 并监控其占比;② 数据无校验器裸奔——`[FSM:G篇 §4.5]` 已裁定校验器是标配不是可选;③ 只有程序员改过这些"配置"——软编码实锤,收回代码
- **不可逆点**:自制配置语言被内容大量使用后,你将永久维护一门语言
- **解药**:表达力需求分级——参数进数据、组合进分类法、逻辑进脚本逃生舱,三层各安其位
- **关联**:卡片 3(它的地基);`[FSM:G篇]`(它的最大实战场)

## 卡片 5:原型(场景实例化)

- **粒度层 / 吸收状态**:对象 / 🟢(`PackedScene.instantiate()` 即原型模式的引擎完全体——以配置好的模板克隆新实例)
- **解决的问题**:实例的初始形态复杂(节点树 + 配置),以模板克隆代替代码逐步构造
- **Godot 形态**:PackedScene + 场景继承。与卡片 3 的标准搭配:**场景管形,def 管数**(同一个 building.tscn,实例化后注入不同 BuildingDef)
- **适用信号**:天天在用,无需信号;真正的决策在翻面

──────── 翻面 ────────

- **入口诱惑**:"继承一下这个场景改两下就是新东西"——场景继承和类继承一样顺手
- **腐化信号**:① 场景继承链 ≥3 层——类继承深渊的场景版,同病同解(拉平为"基础场景 + def 差异化",即卡片 3 搭配);② 用 `duplicate()` 克隆**运行时**对象当原型——把脏状态(当前血量、进行中的 tween)一起复制,原型必须是干净模板(PackedScene 或专门的出厂 def);③ Resource 的 duplicate 语义误区:默认浅复制,嵌套 Resource 仍共享——`[FSM:C篇 §4.4]` 共享引用陷阱的又一入口,需要深复制时显式 `duplicate(true)` 并写明理由
- **不可逆点**:场景继承深链上的中层修改会静默波及全部后代——和类继承一样,链越深,改动越像抽积木
- **解药**:深链拉平;运行时克隆改走模板;共享语义显式化
- **关联**:卡片 3(形数分离搭档);`[FSM:C篇 §4.4]`

## 6. 反面专题:God Object 解剖与拆除

前五册与本册的翻面里,God Object 以不同马甲反复出场,值得一次集中解剖。**三条成因路径**(对应三种手术入口):

| 路径 | 形成过程 | 已建档的马甲 |
|---|---|---|
| 引力阱型 | 协调者/Manager 类不断吸入"不知道放哪"的逻辑 | P1 卡片 4 中介者腐化、P3 卡片 5 组合根膨胀 |
| 便利型 | "反正 player.gd/GameManager 都能摸到,先写这儿" | P3 卡片 1 单例地狱(全局可达是便利型的燃料) |
| 马甲型 | 名字自带正当性的容器悄悄膨胀 | P3 卡片 3 ctx 万物袋、本册卡片 2 翻面①(切碎了的 God Object) |

**拆除手术**(与 P3 §6 同为增量方案):

1. **聚类**:按"变更理由"给巨类的方法分组(`[边界篇 §1.1]` 变更同频)——工具:翻提交历史,总在同一批提交里被改的方法是一组
2. **锚定去处**:每组对应一个目标归属(某系统 / 某组件 / 某策略 / def 数据)——用本册 §7 速查表裁决
3. **先斩状态**:与 P3 手术同理,字段先于方法迁移;字段留转发属性过渡
4. **逐组搬迁**:每组一批提交,巨类方法改为对新归属的委托(保持外部调用兼容),外部调用逐批改直连,清空后删除委托
5. **立碑**:巨类瘦身到只剩单一职责后重命名——旧名字(XXXManager)本身就是引力阱,名字不换,逻辑还会回来

**继承深渊**的拆法是同一手术的纵向版:自最深的叶子类开始,把每层的差异翻译为"组件 or 策略 or def 字段"(§7 表),翻译完一层砍一层,直到回到 `[SOLID篇]` 的一层上限。

**字符串类型编程与魔法数字**(小反模式,一并立案):裸字符串当标识(`if state == "actve"` ——拼错编译器不救你)与裸数字当语义(`if hp < 37`)。Godot 高发区:节点路径、动画名、input action 名、group 名。解药一贯:StringName 常量集中定义(`[FSM:C篇 §4.4]` 已因迁移表需要立过此规)、阈值进 def(`[FSM:A篇 §4.2]` 窗口参数同款)、编辑器引用用 `@export` 而非硬编码路径。

## 7. 裁决速查表:差异的形状 → 承载物

| 变体之间差的是 | 承载物 | 卡片 |
|---|---|---|
| 常数/数值/引用 | def(类型对象) | 3 |
| 算法整体 | 策略 | 1 |
| 能力的集合 | 组件组合 | 2 |
| 内容侧批量生产的成套东西 | 数据驱动(def + 分类法 + 解释器) | 4 |
| 单一定制点、罕见 | Callable 注入 | P1 卡片 5 |
| 类别标签,行为几乎全同 | enum + match | `[FSM索引 §3.1 第1档]` |
| 初始形态(节点树构成) | 场景原型 | 5 |

误配的代价不对称:**把数据形状的差异做成代码(策略类爆炸)贵在生产,把代码形状的差异做成数据(自制配置语言)贵在维护**——后者远更致命,拿不准时先偏代码一侧,等差异的形状显形再右移。

## 8. AI 代理协作约定(供 CLAUDE.md 摘录)

1. 实现任何"多变体"需求前,先用 §7 表判定差异形状并记录;新增策略类时必须证明差异非参数可表达。
2. def(类型对象)中禁止行为开关 bool;行为差异一律经策略字段或封闭 tag。实例只持 def 引用,拷贝 def 数据即为实现错误。
3. 组件间通信一律经宿主;组件代码中出现对兄弟组件的直接引用不予通过。
4. 新建任何配置格式必须同时交付校验器;配置中出现嵌套逻辑模拟(循环/变量语义)时停止实现,上报"进分类法 vs 走脚本逃生舱"的选择建议。
5. 裸字符串标识与魔法数字不予通过;StringName 常量集中定义,阈值归 def。
6. 发现 God Object 征兆(类超 300 行且方法分属多个变更理由)时按 §6 手术方案给出拆除计划,不顺手扩大。
